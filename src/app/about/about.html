Weight Management Medication Prioritization

Proof of Concept Application

This proof of concept application implements clinical decision support (CDS) recommendations for prioritizing patient access to weight management medications via eligibility and prioritization tiering based on clinical guidelines and criteria. It showcases how arbitrary guidelines can be made to support interactive applications so long as they can be represented in computable format using Clinical Quality Language (CQL) and execute over FHIR patient data. It implements the CQL-with-FHIR Implementation Guide (IG).

Primary Clinical Criteria

  • BMI Assessment: Multiple BMI ranges (27-30, 30-35, 35-40, 40+)
  • Cardiovascular Conditions: Ischemic heart disease, hypertension
  • Weight-Related Conditions: Obesity and related comorbidities
  • Heart Function: Left ventricular ejection fraction (HFpEF)

FHIR Data Sources

  • Observations: BMI measurements, lab values
  • Conditions: Active and confirmed diagnoses
  • Medications: Current and historical prescriptions
  • Encounters: Clinical visits and procedures
Clinical Guidelines Reference

This application is based on Department of Veterans Affairs (VA) Scarce Resource Allocation Guidance for Weight Management Medications guidelines. Please refer to the complete documentation available in the Documentation menu for detailed clinical criteria and implementation guidance.

Technical Architecture

CQL-with-FHIR IG

This application implements the CQL-with-FHIR Implementation Guide, which provides a standardized approach for expressing clinical logic using Clinical Quality Language (CQL) and accessing patient data through FHIR R4 resources.

  • FHIR R4 Resource Access
  • Standardized Clinical Logic
  • Interoperable Data Exchange

Key Components

  • HAPI FHIR Server: FHIR R4 resource server
  • CQL Engine: Clinical Quality Language processor
  • Angular Frontend: User interface and visualization
  • Patient Data: Synthetic FHIR resources

Implementation Details

CQL Library Structure

The WeightManagement CQL library includes:

  • BMI calculation and categorization functions
  • Cardiovascular condition assessment
  • Weight-related condition evaluation
  • Heart failure with preserved ejection fraction (HFpEF) detection
  • Patient demographic and clinical data extraction

FHIR Resources Used

  • Patient: Demographics and basic information
  • Observation: BMI measurements, lab values
  • Condition: Diagnoses and clinical status
  • MedicationRequest: Prescription information
  • Encounter: Clinical visits and encounters
Development Status

This is an experimental proof of concept developed by Arizona State University. The application demonstrates the potential of CQL-with-FHIR IG for clinical decision support but should not be used for actual clinical decision-making without proper validation and regulatory approval.
